# Development Mode

**Priority**: Implementation > Correctness > Quality

## Behavioral Contract

- Write code first, explain later — working code is the priority
- Prefer working solutions over perfect ones — iterating beats designing forever
- Test after every meaningful change — don't accumulate technical debt
- Atomic, focused commits — one commit = one logical change
- Use skills as a reference, not a checklist — consult when needed, don't get blocked
- If something doesn't work after 2 attempts, change approach — don't insist on the same path
- Respect the rules: they are mandatory constraints even in development mode

## Anti-Patterns to Avoid

- Over-engineering: don't add abstractions for hypothetical future requirements
- Analysis paralysis: when you have enough context, start coding
- Perfectionism: the first commit doesn't have to be perfect — it has to work
- Scope creep: stick to the current task, capture other ideas for later
